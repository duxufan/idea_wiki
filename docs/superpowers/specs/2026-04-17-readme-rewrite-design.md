# README 重写设计文档

日期：2026-04-17  
状态：待用户审阅

## 目标

重写 `idea_wiki` 的 `README.md`，使其不再像普通仓库说明书，而更像一个清晰的中文方法论文档：

- 先解释“为什么想法与学习必须结合”
- 再解释这套框架如何支持 `idea -> 结构化 -> 调研 -> 对齐 -> 沉淀 -> 新 idea`
- 最后再落到仓库结构、技能与使用方式

新版 README 需要更适合向第一次接触该项目的人解释这套方法，尤其是想理解“idea 驱动研究”工作流的人。

## 已确认需求

### 1. 主要受众

README 的主要受众不是一般 GitHub 路人，也不是只想复制模板的人，而是：

- 想理解“idea 驱动研究”方法论的人
- 想理解如何让个人想法与学界已有工作逐步对齐的人

### 2. 核心主张

README 需要明确表达以下主张：

1. 只有知识积累，不足以推进自己的问题。
2. 只有想法积累，不足以形成可靠研究。
3. 真正需要的是一个闭环，让 idea 驱动学习，让学习反过来修正 idea。
4. AI 的作用不是替人思考，而是把人的模糊想法变成可研究对象，并帮助其与现有工作对齐。

### 3. README 的主结构

README 采用“前半方法论、后半工作流”的混合结构：

1. 标题 + 一句话定位
2. Why this exists / 为什么需要这个框架
3. 这套 workflow 在做什么
4. AI 在里面扮演什么角色
5. 仓库结构如何承载这一闭环
6. 内置技能如何进入流程
7. 这不是什么
8. 如何开始

### 4. 语言与风格

- 使用中文
- 比当前 README 更像方法论文章，而不是普通项目说明
- 保持逻辑强、段落清楚、术语稳定
- 允许加入少量 emoji，提高可读性和 GitHub 首页辨识度
- emoji 只作为节奏与层次辅助，不应变成花哨装饰

## README 的叙事顺序

新版 README 应按以下叙事展开：

### 第一层：必要性

先回答：

> 为什么“想法”和“学习/研究”必须放在一起管理？

要强调两种断裂：

- **只有知识，没有问题主线**：容易积累很多资料，但没有自己的研究推进。
- **只有想法，没有外部校正**：容易停留在主观直觉，难以判断与已有工作的位置关系。

由此引出本项目的目标：

> 把人的原始想法、外部研究材料、概念沉淀、问题空间放进同一个研究闭环里。

### 第二层：工作流

明确给出主线：

`idea -> structure -> research -> alignment -> synthesis -> new idea`

并解释每一段的职责：

- **idea**：人的初始直觉、问题意识、猜想、方向感
- **structure**：把模糊表达整理成可研究的问题空间
- **research**：调入已有工作、资料与外部视角
- **alignment**：让人的表达与学界现有语境对齐
- **synthesis**：形成稳定理解、概念与问题页
- **new idea**：从沉淀中再产生新的问题与方向

### 第三层：AI 的位置

README 需要明确：

AI 在这里不是“自动写摘要的工具”，而是研究对齐器。

它帮助完成：

- 将模糊想法整理为可研究对象
- 找出需要补充的知识与材料
- 将个人表述映射到现有研究语言
- 识别人与学界已有认知之间的偏差、空白与错位

### 第四层：仓库结构

只有在完成前面三层之后，才进入文件结构解释。

每个目录的说明都要服务于 workflow，而不是只是列目录：

- `raw/`：保存外部材料原文，确保原始输入不被污染
- `wiki/sources/`：把来源转成可复用的 source notes
- `wiki/concepts/`：沉淀跨来源也成立的稳定概念
- `wiki/problems/`：保存问题空间、knowledge gaps 与 `idea_seeds`
- `skills/`：让 brainstorming / research intake / source study 进入闭环

## 推荐的 README 结构草案

新版 README 应包含以下一级标题：

1. `# Idea Wiki`
2. `## 为什么要把想法和学习放在一起`
3. `## 这套框架在做什么`
4. `## AI 在这里扮演什么角色`
5. `## 这个仓库如何承载这个闭环`
6. `## 内置技能`
7. `## 这不是什么`
8. `## 如何开始`

## Emoji 使用规则

允许使用少量 emoji，但需要有节制。建议每个一级重点区块最多 1 个：

- `💡`：idea / intuition / problem framing
- `📚`：research / source intake
- `🧭`：alignment / orientation / workflow
- `🧱`：repository structure
- `🤖`：AI role
- `🚫`：non-goals
- `🚀`：getting started

避免：

- 连续堆砌 emoji
- 在每个 bullet 前都加 emoji
- 使用与内容无关的装饰性 emoji

## 边界与非目标

新版 README 仍应明确：

- 这不是传统笔记仓库
- 这不是任务管理器
- 这不是“自动收集一切资料”的 archive
- 这不是让 AI 代替人形成判断

## 成功标准

README 重写完成后，应满足：

1. 第一屏就能让读者理解：这不是普通知识库，而是 idea-driven research workflow。
2. 读者能理解“为什么想法与学习必须结合”。
3. 读者能理解 AI 在其中是“研究对齐器”，而不是普通摘要器。
4. 仓库结构说明从属于 workflow，而不是喧宾夺主。
5. 中文表达更自然，逻辑更强，可读性优于当前 README。
6. emoji 存在但不过量。
