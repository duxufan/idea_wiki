# Linking Rules

## When to create a link

- Create a link when a source materially supports, challenges, refines, or contextualizes another page.
- Do not create links just because two notes use similar vocabulary.

## How to explain a link

- Each source note must include a `Link Reasoning` section.
- Each link rationale should say what kind of relationship exists:
  - support
  - contrast
  - refinement
  - scope guardrail
  - knowledge-gap trigger

## Storage rule

- Explain link reasoning close to the source note where the new link is introduced.
- Problem-space pages may summarize the rationale again if it helps browseability.

## Explainability check

- A fresh reader should be able to answer "Why does this page link there?" without opening chat logs.
